# Лабораторная работа №2

## Тема работы

Реализация паттерна проектирования «Декоратор».

## Цель работы

Изучить применение паттерна «Декоратор» на практике, научиться использовать абстрактные классы и интерфейсы в Python, а также реализовать преобразование данных в различные форматы без изменения основного компонента программы.

## Постановка задачи

Необходимо было реализовать программу, которая получает курсы валют с сайта Центрального банка Российской Федерации и позволяет выводить полученные данные в нескольких форматах:

* JSON;
* YAML;
* CSV.

Также требовалось:

* использовать паттерн «Декоратор»;
* реализовать интерфейс с помощью `ABC` и `@abstractmethod`;
* добавить аннотации типов;
* написать тесты;
* оформить код согласно требованиям PEP-8 и PEP-257.

## Используемый паттерн

В данной работе используется паттерн проектирования **«Декоратор»**.

Основная идея паттерна заключается в том, что дополнительное поведение объекту можно добавлять динамически, не изменяя его исходный код.

В качестве базового компонента используется класс `CurrencyComponent`, который получает данные о курсах валют.

Поверх него могут быть наложены декораторы:

* `JsonDecorator`;
* `YamlDecorator`;
* `CsvDecorator`.

Каждый декоратор получает данные от базового компонента и преобразует их в необходимый формат.

В программе реализованы следующие классы.

## Реализация интерфейса

Для создания общего интерфейса используется модуль `abc`.

Абстрактный класс `Component` содержит метод `operation()`, который обязаны реализовать все наследники.

```python
from abc import ABC, abstractmethod

class Component(ABC):

    @abstractmethod
    def operation(self):
        pass
```

Также был реализован интерфейс `SaveMixin`, который определяет метод сохранения данных в файл.

```python
class SaveMixin(ABC):

    @abstractmethod
    def save_to_file(self, filename: str) -> None:
        pass
```

## Получение данных

Класс `CurrencyComponent` выполняет запрос к API Центрального банка РФ и возвращает данные в виде словаря Python.

```python
response = requests.get(url, timeout=10)
response.raise_for_status()

return response.json()
```

После получения данные могут быть обработаны различными декораторами.

---

## JSON-декоратор

Класс `JsonDecorator` преобразует словарь Python в JSON-строку.

```python
return json.dumps(
    data,
    ensure_ascii=False,
    indent=4
)
```

Параметр `ensure_ascii=False` используется для корректного отображения русских символов.

Также реализован метод сохранения результата в файл формата `.json`.

---

## YAML-декоратор

Класс `YamlDecorator` преобразует данные в формат YAML.

```python
return yaml.safe_dump(
    data,
    allow_unicode=True,
    sort_keys=False
)
```

Параметр `allow_unicode=True` позволяет корректно отображать кириллические символы.

Также реализовано сохранение результата в файл формата `.yaml`.

---

## CSV-декоратор

Класс `CsvDecorator` формирует CSV-таблицу с информацией о валютах.

В CSV-файл записываются:

* код валюты;
* название валюты;
* номинал;
* текущий курс;
* предыдущий курс.

Пример формирования строки:

```python
writer.writerow([
    code,
    currency.get("Name"),
    currency.get("Nominal"),
    currency.get("Value"),
    currency.get("Previous")
])
```

Результат может быть сохранён в файл формата `.csv`.
