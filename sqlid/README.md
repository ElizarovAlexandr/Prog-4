# Лабораторная работа: SOLID

## Тема

Пример нарушения и исправления принципа **Dependency Inversion Principle**.

## Цель работы

Цель работы — рассмотреть один из принципов SOLID, показать пример его нарушения и затем исправить код так, чтобы он соответствовал выбранному принципу.

---

## Выбранный принцип

Для работы был выбран принцип **DIP — Dependency Inversion Principle**, то есть принцип инверсии зависимостей.

Данный принцип говорит о том, что модули верхнего уровня не должны зависеть от модулей нижнего уровня. Оба должны зависеть от абстракций.

Проще говоря, основной класс программы не должен быть жестко привязан к конкретной реализации другого класса.

---

## Предметная область

В качестве предметной области была выбрана система отправки уведомлений.

Например, интернет-магазин должен отправлять пользователю уведомление о заказе. Уведомление может быть отправлено разными способами:

- по email;
- по SMS;
- через мессенджер.

## Пример нарушения принципа

В плохом примере класс `NotificationServiceBad` напрямую создает объект класса `EmailSenderBad`.

```python
class EmailSenderBad:
    def send_email(self, message: str) -> str:
        return f"Email отправлен: {message}"


class NotificationServiceBad:
    def __init__(self) -> None:
        self.sender = EmailSenderBad()

    def notify(self, message: str) -> str:
        return self.sender.send_email(message)
