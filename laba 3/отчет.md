# Лабораторная работа №3

## Разработка REST API для отслеживания курсов валют

### Цель работы

Целью лабораторной работы является разработка REST API с использованием FastAPI и SQLAlchemy для отслеживания курсов валют Центрального банка Российской Федерации.

Приложение позволяет создавать пользователей, подписывать их на интересующие валюты, получать актуальные курсы валют и хранить данные в базе SQLite.

---

## Используемые технологии

В ходе выполнения работы были использованы следующие технологии:

* FastAPI;
* SQLAlchemy ORM;
* SQLite;
* aiosqlite;
* Pydantic;
* httpx;
* XML Parser из стандартной библиотеки Python.

## Описание базы данных

Для хранения информации используется база данных SQLite.

В проекте реализованы три основные сущности.

### Пользователь (User)

Содержит информацию о зарегистрированных пользователях.

Поля:

* id;
* username;
* email;
* created_at.

### Валюта (Currency)

Содержит данные о валютах, полученных с сайта Центрального банка РФ.

Поля:

* id;
* code;
* name;
* rate.

### Подписка (Subscription)

Используется для связи пользователей и валют.

Поля:

* user_id;
* currency_id.

Один пользователь может подписываться на несколько валют, а одна валюта может отслеживаться несколькими пользователями.

---

## Получение данных о валютах

Информация о курсах валют загружается с официального сервиса Центрального банка РФ:

```text
https://www.cbr.ru/scripts/XML_daily.asp
```

Для выполнения запросов используется библиотека `httpx`.

Полученный XML-документ обрабатывается с помощью модуля `xml.etree.ElementTree`.

После обработки данные сохраняются в базу данных.

## Реализованные эндпоинты

### Пользователи

Создание пользователя:

```http
POST /users/
```

Получение списка пользователей:

```http
GET /users/
```

Получение информации о пользователе:

```http
GET /users/{user_id}
```

Обновление данных пользователя:

```http
PUT /users/{user_id}
```

Удаление пользователя:

```http
DELETE /users/{user_id}
```

---

### Валюты

Получение списка валют:

```http
GET /currencies/
```

Обновление данных о валютах с сайта ЦБ РФ:

```http
POST /currencies/update
```

Получение текущего курса валюты:

```http
GET /currencies/{currency_code}/rate
```

---

### Подписки

Создание подписки:

```http
POST /subscriptions/
```

Удаление подписки:

```http
DELETE /subscriptions/
```

---

## Валидация данных

Для проверки входных данных используются Pydantic-схемы.

В частности:

* выполняется проверка формата электронной почты;
* проверяется наличие обязательных полей;
* выполняется преобразование данных в необходимые типы.

---

## Обработка ошибок

В приложении реализована обработка основных ошибок:

* 404 Not Found — пользователь или валюта не найдены;
* 409 Conflict — попытка создать уже существующую подписку;
* ошибки валидации входных данных;
* ошибки получения данных от внешнего API.
